Chapter 6: Dynamic URLs & Single Post Pages

This chapter covers how to route users from a main list of posts to dedicated, dynamically generated pages for individual posts using Django slugs.

1 Updating Post Data in the Admin Panel

Start your server and go to the Django admin panel (localhost:8000/admin).

Navigate to your existing posts and ensure every post has a completed body and slug field (e.g., my-first-post). The slug will be used to generate a unique URL for each post page.

2 Setting up App Names and Named URLs

Open urls.py inside the posts app and add app_name = 'posts' at the top to declare a namespace.

from django.urls import path
from . import views

app_name = 'posts' # App namespace

urlpatterns = [
    # Added name parameter to the list path
    path('', views.post_list, name='list'), 
]

Open your master layout.html template and update the navigation links to use Django's template tags instead of hardcoded paths.

<!-- Inside layout.html -->
<a href="{% url 'posts:list' %}">Posts</a>

3 Creating a URL Path Converter for Slugs

Back in your posts app urls.py, add a new path for individual post pages using a slug path converter. This tells Django to expect a dynamic slug string in the URL and pass it to the view.

urlpatterns = [
    path('', views.post_list, name='list'),
    # New path converter for the slug
    path('<slug:slug>', views.post_page, name='page'), 
]

4 Defining the Single Post View

Open views.py inside the posts app and create a new function called post_page that takes request and slug as parameters. Fetch the specific post and render the template.

from django.shortcuts import render
from .models import Post

def post_page(request, slug):
    # Fetch the single specific post by matching the slug
    post = Post.objects.get(slug=slug)
    
    # Return rendered template, passing the post data in a dictionary
    return render(request, 'posts/post_page.html', {'post': post})

5 Creating the Post Page Template

Inside your nested templates/posts directory, create a new file named post_page.html. Extend your master layout and output the specific post's data.

{% extends 'layout.html' %}

{% block content %}
    <article>
        <h1>{{ post.title }}</h1>
        <small>{{ post.date }}</small>
        <p>{{ post.body }}</p>
    </article>
{% endblock %}

6 Linking the Post List to Individual Pages

Open your existing post_list.html template. Find where you output the post titles and wrap the title in an anchor tag, passing the specific post's slug dynamically to the URL tag.

<!-- Inside post_list.html -->
<h2>
    <a href="{% url 'posts:page' slug=post.slug %}">
        {{ post.title }}
    </a>
</h2>

Note: Visit your live site; you should now be able to click on any post title in the list and be routed to a dedicated URL page specifically for that post!
