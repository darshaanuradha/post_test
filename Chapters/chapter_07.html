Chapter 7: Adding and Handling Images

This chapter covers how to configure Django to handle user-uploaded media files, update your database models, and display dynamic images inside your templates.

1 Configuring Media Settings

Open the settings.py file inside your top-level project folder. Scroll to the bottom of the file (near static files) and add two new variables to tell Django where to store uploaded images.

import os # Make sure os is imported at the top of the file

# Add these at the bottom of settings.py
MEDIA_URL = 'media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

2 Updating Main Project URLs

Open the master urls.py file in your main project folder. Add the necessary imports and append the media paths to your urlpatterns.

from django.contrib import admin
from django.urls import path, include
# Add these two new imports:
from django.conf.urls.static import static
from django.conf import settings

urlpatterns = [
    path('admin/', admin.site.urls),
    path('posts/', include('posts.urls')),
] + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT) # Append here

3 Installing the Pillow Library

Django requires an external library called Pillow to handle image fields. Ensure your virtual environment is activated and open the VS Code terminal.

pip install pillow

4 Updating the Post Model

Open models.py inside your posts app. Add a new field to your Post class for a banner image.

from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=75)
    body = models.TextField()
    slug = models.SlugField()
    date = models.DateTimeField(auto_now_add=True)
    # Add the new ImageField:
    banner = models.ImageField(default='fallback.png', blank=True)

    def __str__(self):
        return self.title

The blank=True argument makes uploading an image optional, while default='fallback.png' ensures there is a default image displayed if the user doesn't provide one.

5 Applying New Migrations

Because you altered your model, you must update the database schema to reflect the new image column. Run these commands in your terminal:

py manage.py makemigrations
py manage.py migrate

6 Uploading Images via the Admin Panel

Start your server with py manage.py runserver and navigate to your Django admin panel (localhost:8000/admin).

Edit your existing posts. You will now see a new "Banner" field where you can choose a file from your computer and save it to the post.

Note: Django will automatically generate a new media folder in your VS Code file tree once the first image is uploaded. Drag and drop a placeholder image named fallback.png into that new media folder so your default fallback works properly.

7 Displaying Images in Templates

Open your post_page.html template located in your posts app. Add an <img> element and set the source attribute dynamically using Django's template engine.

{% extends 'layout.html' %}

{% block content %}
    <article>
        <!-- Output the banner image -->
        <img src="{{ post.banner.url }}" alt="{{ post.title }}" class="banner-image">
        
        <h1>{{ post.title }}</h1>
        <small>{{ post.date }}</small>
        <p>{{ post.body }}</p>
    </article>
{% endblock %}
