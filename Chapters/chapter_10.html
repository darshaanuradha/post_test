1. Setting up the Logout URL and View

Open urls.py in your users app and set up the logout path. Then, create the view to handle the logic.

users/urls.py

from django.urls import path
from . import views

app_name = 'users'

urlpatterns = [
    # ... your other paths ...
    path('logout/', views.logout_view, name='logout'),
]

users/views.py

from django.shortcuts import render, redirect
from django.contrib.auth import logout

def logout_view(request):
    if request.method == 'POST':
        logout(request)
        return redirect('posts:list')

2. Adding a Logout Button to the Navigation

Because logging out should be done via a POST request for security, you must use a form rather than a standard link. Add an HTML form element to your navigation menu in your master layout.

templates/layout.html

<!-- Inside your navigation menu -->
<form action="{% url 'users:logout' %}" method="POST" style="display:inline;">
    {% csrf_token %}
    <button type="submit">Log out</button>
</form>

3. Creating a "New Post" Route and Template

Ensure you place the new-post path above your slug path, or Django will mistakenly try to interpret 'new-post' as a post slug.

posts/urls.py

from django.urls import path
from . import views

app_name = 'posts'

urlpatterns = [
    # ... other paths ...
    path('new-post/', views.post_new, name='new-post'),
    path('<slug:slug>/', views.post_detail, name='post-detail'),
]

posts/views.py

from django.shortcuts import render

def post_new(request):
    return render(request, 'posts/post_new.html')

posts/templates/posts/post_new.html

{% extends 'layout.html' %}

{% block content %}
    <h1>Create a New Post</h1>
    <!-- Future form will go here -->
{% endblock %}

4. Protecting Pages with the Login Required Decorator

To ensure only registered users can access the new post page, add the @login_required decorator to your view.

posts/views.py (Updated)

from django.shortcuts import render
from django.contrib.auth.decorators import login_required

@login_required(login_url="/users/login/")
def post_new(request):
    return render(request, 'posts/post_new.html')

5. Handling the "Next" Redirect Parameter

When a user is redirected by the decorator, Django appends a ?next= parameter to the URL to remember where they were trying to go. Pass this through your login form.

users/templates/users/login.html

<form action="{% url 'users:login' %}" method="POST">
    {% csrf_token %}
    <!-- Your username and password inputs go here -->

    {% if request.GET.next %}
        <input type="hidden" name="next" value="{{ request.GET.next }}">
    {% endif %}
    
    <button type="submit">Log in</button>
</form>

users/views.py (Login View Update)

# Inside your login_view, after successful authentication...
# login(request, user)

if 'next' in request.POST:
    return redirect(request.POST.get('next'))
else:
    return redirect('posts:list')

6. Conditionally Displaying Navigation Links

Dynamically update the navigation bar based on the user's login status using Django template conditionals.

templates/layout.html

<nav>
    <a href="{% url 'posts:list' %}">Home</a>

    {% if user.is_authenticated %}
        <a href="{% url 'posts:new-post' %}">New Post</a>
        
        <form action="{% url 'users:logout' %}" method="POST" style="display:inline;">
            {% csrf_token %}
            <button type="submit">Log out</button>
        </form>
        
    {% else %}
        <a href="{% url 'users:register' %}">User Registration</a>
        <a href="{% url 'users:login' %}">Login</a>
        
    {% endif %}
</nav>
