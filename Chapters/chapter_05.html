Chapter 5: Django Admin & Dynamic Templates

A guide to utilizing the Django Admin panel and displaying database records on your web pages.

1 Accessing the Admin Panel

Activate your virtual environment and start your server using:

py manage.py runserver

Open your browser and navigate to localhost:8000/admin to see the default Django Administration login page.

2 Creating a Superuser

To log in to the admin panel, you need an administrative account. Stop your server in the terminal by pressing Ctrl + C.

Run the following command:

py manage.py createsuperuser

Follow the prompts to enter a username, an optional email address (you can leave this blank), and a password.

📝 Note: The password must be at least 8 characters long and cannot be a common word.

Restart your server (py manage.py runserver) and log in to the admin panel with your newly created credentials.

3 Registering Models in the Admin Panel

By default, your custom models won't appear in the admin panel. Open the admin.py file located inside your posts app folder.

Import your Post model at the top of the file:

from .models import Post

Register the model so the admin panel knows about it:

admin.site.register(Post)

Save the file and refresh your browser. You will now see your Posts table and can use the admin panel as a Content Management System (CMS) to easily add, edit, and manage your posts.

4 Fetching Data in Views

To display these posts on your actual website, open views.py inside the posts app.

Import your model at the top:

from .models import Post

Inside your post_list function, retrieve the posts from the database and assign them to a variable:

posts = Post.objects.all().order_by('-date')
💡 Tip: The minus sign before 'date' sorts the posts in descending order, putting the newest posts at the top.

Pass this data to your template by providing a dictionary as the third argument in your render function: 

return render(request, 'posts/post_list.html', {'posts': posts})

5 Rendering Dynamic Data in Templates

Open your post_list.html template located in your nested templates/posts folder.

Use Django's template engine to loop through the data dictionary you just passed from the view. Start the loop with:

{% for post in posts %}

Inside the loop, you can output specific fields from each post using double curly braces, such as:

  • {{ post.title }}
  • {{ post.date }}
  • {{ post.body }}

Close the loop with:

{% endfor %}

Visit your live site at localhost:8000/posts/ to see your database information dynamically displayed on the page.
