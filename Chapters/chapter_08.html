Chapter 8: User Registration

This chapter covers how to create a dedicated user management app, utilize Django's built-in authentication forms, and securely handle POST requests to register new users.

1 Creating the Users App

Open your terminal, ensure your virtual environment is activated, and navigate to your myproject directory. Run the following command to create a new module for user management: 

py manage.py startapp users

Open your main project's settings.py file, scroll down to the INSTALLED_APPS list, and register the new app.

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'posts',
    'users', # Add your new app here
]

2 Configuring User URLs

Create a new urls.py file inside your new users app directory to set up your basic URL configurations.

# Inside users/urls.py
from django.urls import path
from . import views

app_name = 'users'

urlpatterns = [
    path('register/', views.register_view, name='register'),
]

Next, open your master project's urls.py file and include your new app's URLs.

# Inside main project urls.py
urlpatterns = [
    path('admin/', admin.site.urls),
    path('posts/', include('posts.urls')),
    path('users/', include('users.urls')), # Append the users paths
] + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

3 & 4 Setting Up the View & Handling Form Logic

Open views.py inside the users app. Import Django's built-in UserCreationForm and the redirect function. Then, build out the logic to handle both GET (viewing the form) and POST (submitting the form) requests.

from django.shortcuts import render, redirect
from django.contrib.auth.forms import UserCreationForm

def register_view(request):
    # Check if the form has been submitted
    if request.method == 'POST':
        # Instantiate form with submitted data
        form = UserCreationForm(request.POST) 
        if form.is_valid():
            form.save() # Save the new user to the database
            return redirect('posts:list') # Redirect to home page
    else:
        # GET request: Instantiate an empty form
        form = UserCreationForm() 
        
    # Return rendered template, passing the form context
    return render(request, 'users/register.html', { 'form': form })

5 & 6 Creating the Registration Template

Inside your users app, create a templates/users/ directory, and create a new file named register.html. Extend your master layout and build the HTML form.

Crucial Security Step: You must include the {% csrf_token %} tag inside your form to prevent Cross-Site Request Forgeries. Without it, Django will reject the submission and throw an error. 
{% extends 'layout.html' %}

{% block content %}
    <h1>Register for an Account</h1>
    
    <!-- Action points back to the register URL -->
    <form action="/users/register/" method="POST">
        {% csrf_token %}
        
        <!-- Output dynamic form fields -->
        {{ form }}
        
        <button type="submit">Register</button>
    </form>
{% endblock %}

7 Updating the Navigation Bar

Finally, open your master layout.html file. Add a new navigation link to the menu so users can easily find the registration page using Django's URL template tag.

<!-- Inside layout.html navigation menu -->
<a href="{% url 'posts:list' %}">Home</a>
<a href="{% url 'users:register' %}">Register</a>
