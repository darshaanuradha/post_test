Chapter 9: User Login & Auto-Login

This chapter covers how to create a secure user login system using Django's built-in authentication forms, and how to improve user experience by auto-logging users in immediately after they register.

1 Updating the Navigation Bar

Open your master layout.html template. Copy the navigation link you made for user registration and paste it directly underneath to create a new link for the login page.

<!-- Inside layout.html navigation menu -->
<a href="{% url 'posts:list' %}">Home</a>
<a href="{% url 'users:register' %}">Register</a>
<!-- New Login Link -->
<a href="{% url 'users:login' %}">User Login 🔒</a>

2 Configuring Login URLs

Open the urls.py file inside your users app. Add a new path for the login view.

# Inside users/urls.py
urlpatterns = [
    path('register/', views.register_view, name='register'),
    # New Login Path
    path('login/', views.login_view, name='login'),
]

3 & 4 Setting Up the Login View & Form Logic

Open views.py inside the users app. Import the built-in AuthenticationForm and the login function. Then, create the login_view to handle POST and GET requests.

Note: Unlike the registration form, the authentication form requires you to explicitly name the data argument: form = AuthenticationForm(data=request.POST). 
from django.shortcuts import render, redirect
from django.contrib.auth.forms import UserCreationForm, AuthenticationForm
from django.contrib.auth import login

def login_view(request):
    if request.method == 'POST':
        # Explicitly pass data argument
        form = AuthenticationForm(data=request.POST)
        if form.is_valid():
            # Retrieve the authenticated user
            user = form.get_user()
            # Log the user in
            login(request, user)
            return redirect('posts:list')
    else:
        # GET request: empty form
        form = AuthenticationForm()
        
    return render(request, 'users/login.html', { 'form': form })

5 Creating the Login Template

Inside your users/templates/users directory, copy your register.html file and name the new file login.html. Update the title, heading, and form action.

{% extends 'layout.html' %}

{% block content %}
    <h1>User Login</h1>
    
    <!-- Update the action URL -->
    <form action="{% url 'users:login' %}" method="POST">
        {% csrf_token %}
        {{ form }}
        <button type="submit">Login</button>
    </form>
{% endblock %}

6 Auto-Login on Registration

To prevent users from having to log in manually right after creating an account, update your existing register_view function in views.py. Because form.save() returns the created user object, you can pass it directly into the login function.

# Inside views.py -> update the existing register_view

def register_view(request):
    if request.method == 'POST':
        form = UserCreationForm(request.POST) 
        if form.is_valid():
            # NEW: Log the user in immediately using the saved user object
            login(request, form.save()) 
            return redirect('posts:list')
    else:
        form = UserCreationForm() 
        
    return render(request, 'users/register.html', { 'form': form })
