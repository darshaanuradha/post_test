Django Project Setup Guide

1. Prerequisites & Initial Setup

  • Ensure you have learned the basics of Python and have a code editor installed, such as Visual Studio Code (VS Code).
  • Ensure you have Python installed (can be downloaded from python.org).
  • Create a new folder for your project (e.g., lesson one) and open it in VS Code.
  • Open a terminal window in VS Code using Ctrl + ` (control and the backtick).
  • Verify your Python installation by checking the version. Type py --version on Windows, or python3 --version on Mac/Linux.

2. Virtual Environment & Django Installation

  • Create a virtual environment by typing py -m venv .venv (or python3 instead of py).
  • Activate the virtual environment. On Windows, type source .venv/scripts/activate. On Mac or Linux, type source .venv/bin/activate. You will see (.venv) appear in the terminal prompt when successful.
  • Install Django by typing py -m pip install Django (use a capital "D"). You may also optionally upgrade pip if prompted.
  • Verify the installation by opening the Python REPL (type py), then typing import django followed by print(django.__version__). Exit the REPL by typing quit().

3. Creating the Project & Running the Server

  • Create the project by typing django-admin startproject myproject (replace myproject with your desired name).
  • Navigate into your newly created project folder by typing cd myproject.
  • Start the server to ensure everything is working by typing py manage.py runserver.
  • Open a web browser and go to localhost:8000 (or http://127.0.0.1:8000) to see the default Django success page.
  • Stop the server in the terminal by pressing Ctrl + C.

4. Configuring Views and URLs

  • Open the urls.py file inside the inner myproject folder. Add new empty paths to the urlpatterns list for your homepage (path('', ...)) and an about page (path('about/', ...)).
  • Create a new file called views.py in the same directory.
  • Inside views.py, add from django.http import HttpResponse. Create two functions, homepage and about, that return simple text messages using HttpResponse.
  • Go back to urls.py and import the views using from . import views. Link the views to the paths you created by adding views.homepage and views.about to the respective path patterns.

5. Creating HTML Templates

  • At the top level of your myproject folder (alongside the manage.py file), create a new directory named templates.
  • Inside this directory, create two HTML files: home.html and about.html. Add basic HTML content (like <h1> tags) and use an anchor tag (<a>) in each file to link to the other page.
  • Open settings.py (located in the inner myproject folder), scroll down to the TEMPLATES section, and add 'templates' inside the DIRS list so Django knows where to find them.
  • Open views.py and change the import to from django.shortcuts import render. Change the return statements in your functions to use the render function instead of HttpResponse, passing in the request and the template names (e.g., return render(request, 'home.html')).

6. Adding Static Files (CSS and JavaScript)

  • At the top level of your project (alongside the templates directory), create a new folder named static.
  • Inside static, create a css folder containing a style.css file, and a js folder containing a main.js file. Add some basic CSS styles and a simple console.log() statement to test them.
  • Open settings.py and add import os at the very top. Scroll to the bottom and add a new setting: STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')].
  • Finally, update your HTML templates to use the Django templating engine. Add {% load static %} just below the <!DOCTYPE html> declaration.
  • Link your stylesheet in the HTML <head> using {% static 'css/style.css' %} in the href attribute. Link your JavaScript file using a <script defer> tag with {% static 'js/main.js' %} in the src attribute.
