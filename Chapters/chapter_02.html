Django Guide: Apps & Template Inheritance

1. Environment Setup & VS Code Extensions

  • Activate your virtual environment: source .venv/scripts/activate (Windows) or source .venv/bin/activate (Mac/Linux).
  • Navigate into your project folder: cd myproject.
  • Pro Tip: Install the Django VS Code extension for syntax highlighting and VS Code Icons to make your file structure easier to navigate.

2. Creating a Modular App

  • Create a new app named "posts": py manage.py startapp posts.
  • Register the app: Open settings.py and add 'posts' to the INSTALLED_APPS list.

3. Setting up Views and App URLs

  • Create the view: In posts/views.py, define post_list to return render(request, 'posts/post_list.html').
  • Create app-level URLs: Create posts/urls.py and map the empty string '' to your view.
  • Connect to master URLs: In the main urls.py, use include() to link 'posts/' to posts.urls.

4. Namespacing and Creating App Templates

  • Namespace Folders: Create templates/posts/ inside your app folder. This ensures Django doesn't confuse post_list.html with files of the same name in other apps.
  • Create post_list.html inside that nested folder.
  • Emmet Fix: If HTML shortcuts fail, add a mapping for django-html to html in VS Code "Include Languages" settings.

5. Creating a Master Layout Template

  • Create a layout.html in your top-level templates directory to hold shared elements like <head> and <nav>.
  • Load static files: {% load static %} at the very top.
  • Define blocks: Use {% block title %} and {% block content %} to mark where child pages will inject their specific data.

6. Extending the Layout in Other Templates

  • Clean up pages: Remove all boilerplate from home.html, about.html, etc.
  • Inherit the master: Add {% extends 'layout.html' %} to the top of each file.
  • Fill the blocks: Wrap your unique content inside the corresponding {% block %} tags.
