The Complete Django Beginner's Guide

Step-by-step tutorial from setup to authentication and databases.

Chapter 1: Django Project Setup Guide

1. Prerequisites & Initial Setup

  • Ensure you have learned the basics of Python and have a code editor installed, such as Visual Studio Code (VS Code).
  • Ensure you have Python installed (download from python.org).
  • Create a new folder for your project (e.g., lesson_one) and open it in VS Code.
  • Open a terminal window in VS Code using Ctrl + ` (control and the backtick).
  • Verify your Python installation by checking the version:
    Windows: py --version
    Mac/Linux: python3 --version

2. Virtual Environment & Django Installation

Create a virtual environment to keep your dependencies contained:

Bash 
py -m venv .venv

(Use python3 instead of py on Mac/Linux)

Activate the virtual environment. You will see (.venv) appear in the terminal prompt when successful.

  • Windows: source .venv/scripts/activate
  • Mac/Linux: source .venv/bin/activate

Install Django (use a capital "D"):

Bash 
py -m pip install Django

Verify the installation by opening the Python REPL:

Bash 
py

Then type:

Python 
import django
print(django.__version__)
quit()

3. Creating the Project & Running the Server

Create the project (replace myproject with your desired name):

Bash 
django-admin startproject myproject

Navigate into your newly created project folder:

Bash 
cd myproject

Start the server to ensure everything is working:

Bash 
py manage.py runserver

Open a web browser and go to localhost:8000 (or http://127.0.0.1:8000) to see the default Django success page. Stop the server in the terminal by pressing Ctrl + C.

4. Configuring Views and URLs

Open the urls.py file inside the inner myproject folder. Add new empty paths for your homepage and an about page.

Create a new file called views.py in the same directory. Inside views.py, create two functions that return simple text messages:

Python 
from django.http import HttpResponse

def homepage(request):
    return HttpResponse("Welcome to the Home Page")

def about(request):
    return HttpResponse("About Us")

Go back to urls.py, import the views, and link them:

Python 
from django.contrib import admin
from django.urls import path
from . import views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.homepage),
    path('about/', views.about),
]

5. Creating HTML Templates

At the top level of your myproject folder (alongside manage.py), create a new directory named templates.

Inside this directory, create home.html and about.html. Add basic HTML content and an anchor tag <a> in each to link to the other.

Open settings.py (in the inner myproject folder), scroll to the TEMPLATES section, and add 'templates' inside the DIRS list:

Python 
TEMPLATES = [
    {
        ...
        'DIRS': ['templates'],
        ...
    },
]

Open views.py and update it to use the render function:

Python 
from django.shortcuts import render

def homepage(request):
    return render(request, 'home.html')

def about(request):
    return render(request, 'about.html')

6. Adding Static Files (CSS and JavaScript)

At the top level of your project, create a new folder named static. Inside static, create a css folder with a style.css file, and a js folder with a main.js file.

Open settings.py, add import os at the top, scroll to the bottom, and add:

Python 
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

Update your HTML templates to load static files. Add this below <!DOCTYPE html>:

HTML 
{% load static %}

Link your CSS and JS in your HTML <head>:

HTML 
<link rel="stylesheet" href="{% static 'css/style.css' %}">
<script defer src="{% static 'js/main.js' %}"></script>

Chapter 2: Apps & Template Inheritance

1. Environment Setup & VS Code Extensions

Make sure your virtual environment is active and you are inside myproject.

Pro Tip: Install the Django VS Code extension for syntax highlighting and VS Code Icons to make your file structure easier to navigate.

2. Creating a Modular App

Create a new app named "posts":

Bash 
py manage.py startapp posts

Register the app in settings.py by adding 'posts' to the INSTALLED_APPS list.

3. Setting up Views and App URLs

In posts/views.py, define your view:

Python 
from django.shortcuts import render

def post_list(request):
    return render(request, 'posts/post_list.html')

Create posts/urls.py and map the route:

Python 
from django.urls import path
from . import views

urlpatterns = [
    path('', views.post_list),
]

In the main urls.py, use include() to link them:

Python 
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('posts/', include('posts.urls')),
    # ... other paths
]

4. Namespacing and Creating App Templates

Create a templates folder inside your posts app folder, and then a posts folder inside that (posts/templates/posts/). This namespaces your templates. Create post_list.html inside that nested folder.

5. Creating a Master Layout Template

Create layout.html in your top-level templates directory to hold shared elements like <head> and <nav>.

HTML 
{% load static %}
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <nav>
    </nav>
    <main>
        {% block content %}
        {% endblock %}
    </main>
</body>
</html>

6. Extending the Layout in Other Templates

Remove boilerplate from your other HTML files and inherit the master layout:

HTML 
{% extends 'layout.html' %}

{% block title %}Home Page{% endblock %}

{% block content %}
    <h1>Welcome Home</h1>
{% endblock %}

Chapter 3: Django Models & Migrations

1. Creating a Database Model

Note: Models in Django are Python classes that map to tables in your database.

Open models.py inside your posts app and create a model:

Python 
from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=75)
    body = models.TextField()
    slug = models.SlugField()
    date = models.DateTimeField(auto_now_add=True)

2. Applying Migrations

Apply Django's built-in migrations first:

Bash 
py manage.py migrate

Make migrations for your new Post app:

Bash 
py manage.py makemigrations

Apply your new migration to build the table:

Bash 
py manage.py migrate

3. Fetching and Rendering Data

In posts/views.py, fetch the data and pass it to the template:

Python 
from .models import Post

def post_list(request):
    posts = Post.objects.all().order_by('-date')
    return render(request, 'posts/post_list.html', {'posts': posts})

In post_list.html, loop through the data:

HTML 
{% extends 'layout.html' %}

{% block content %}

        <p>{{ post.date }}</p>
        <p>{{ post.body }}</p>
    {% endfor %}
{% endblock %}

Chapter 6: Dynamic URLs & Single Post Pages

1. App Namespaces and URLs

In posts/urls.py, set up named URLs and a path converter for slugs:

Python 
from django.urls import path
from . import views

app_name = 'posts' # App namespace

urlpatterns = [
    path('', views.post_list, name='list'),
    path('<slug:slug>/', views.post_page, name='page'), 
]

2. Defining the Single Post View

In posts/views.py:

Python 
def post_page(request, slug):
    post = Post.objects.get(slug=slug)
    return render(request, 'posts/post_page.html', {'post': post})

3. Creating the Template & Linking

Create posts/templates/posts/post_page.html:

HTML 
{% extends 'layout.html' %}
{% block content %}
    <article>
        <h1>{{ post.title }}</h1>
        <small>{{ post.date }}</small>
        <p>{{ post.body }}</p>
    </article>
{% endblock %}

Update post_list.html to link to the new page:

HTML 
<h2>
    <a href="{% url 'posts:page' slug=post.slug %}">
        {{ post.title }}
    </a>
</h2>

Chapter 8: User Registration

1. Creating the Users App

Bash

Add 'users' to INSTALLED_APPS in settings.py.

Create users/urls.py:

    path('register/', views.register_view, name='register'),
]

Include this in your main urls.py:

Python

3. The Registration View & Template

In users/views.py:

Python 
from django.shortcuts import render, redirect
from django.contrib.auth.forms import UserCreationForm

def register_view(request):
    if request.method == 'POST':
        form = UserCreationForm(request.POST) 
        if form.is_valid():
            form.save()
            return redirect('posts:list')
    else:
        form = UserCreationForm() 
        
    return render(request, 'users/register.html', { 'form': form })

Create users/templates/users/register.html:

HTML 
{% extends 'layout.html' %}
{% block content %}
    <h1>Register</h1>
    <form action="{% url 'users:register' %}" method="POST">
        {% csrf_token %}
        {{ form }}
        <button type="submit">Register</button>
    </form>
{% endblock %}

Chapter 9: User Login & Auto-Login

1. The Login View & URL

In users/urls.py, add path('login/', views.login_view, name='login'),.

In users/views.py:

Python 
from django.contrib.auth.forms import AuthenticationForm
from django.contrib.auth import login

def login_view(request):
    if request.method == 'POST':
        form = AuthenticationForm(data=request.POST)
        if form.is_valid():
            user = form.get_user()
            login(request, user)
            return redirect('posts:list')
    else:
        form = AuthenticationForm()
        
    return render(request, 'users/login.html', { 'form': form })

2. The Login Template

Create login.html (similar to register.html):

HTML 
<form action="{% url 'users:login' %}" method="POST">
    {% csrf_token %}
    {{ form }}
    <button type="submit">Login</button>
</form>

3. Auto-Login on Registration

Chapter 10: Authentication Extras

1. Logout View & Form

In users/urls.py, add path('logout/', views.logout_view, name='logout'),.

In your navigation (layout.html), use a form for logout to ensure security:

2. Login Required & Next Parameters

In posts/views.py, protect the new post route (ensure this URL path is placed above your slug path in urls.py):

Python 
from django.contrib.auth.decorators import login_required

@login_required(login_url="/users/login/")
def post_new(request):
    return render(request, 'posts/post_new.html')

Pass the next parameter in login.html:

HTML 
{% if request.GET.next %}
    <input type="hidden" name="next" value="{{ request.GET.next }}">
{% endif %}

And handle it in the login_view:

Python 
if 'next' in request.POST:
    return redirect(request.POST.get('next'))
else:
    return redirect('posts:list')

Chapter 11: Forms, Views, and Foreign Keys

1. Creating a Custom Form

Create forms.py in your posts app:

Python 
from django import forms
from . import models

class CreatePost(forms.ModelForm):
    class Meta:
        model = models.Post
        fields = ['title', 'body', 'slug', 'banner']

2. The Form Template

In post_new.html, be sure to include enctype for image uploads:

HTML 
<form action="{% url 'posts:new-post' %}" method="POST" enctype="multipart/form-data">
    {% csrf_token %}
    {{ form }}
    <button type="submit">Submit</button>
</form>

3. Adding an Author Foreign Key

In posts/models.py, add a relationship to Django's built-in User model:

Python 
from django.contrib.auth.models import User

class Post(models.Model):
    # ... existing fields
    author = models.ForeignKey(User, on_delete=models.CASCADE, default=None)
Important: Delete any existing posts in the admin panel first, then run makemigrations and migrate.

4. Saving the Post with the Current User

Update post_new in posts/views.py:

Python 
from . import forms

@login_required(login_url="/users/login/")
def post_new(request):
    if request.method == 'POST':
        form = forms.CreatePost(request.POST, request.FILES)
        if form.is_valid():
            newpost = form.save(commit=False)
            newpost.author = request.user
            newpost.save()
            return redirect('posts:list')
    else:
        form = forms.CreatePost()
        
    return render(request, 'posts/post_new.html', {'form': form})

5. Displaying the Author

In post_list.html:

HTML
