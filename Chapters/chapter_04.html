Chapter 4: The Django ORM & Shell

A guide to interacting with your database using Python.

1 Environment Setup & Starting the Shell

Open your terminal in VS Code and activate your virtual environment:

  • Windows: source .venv/scripts/activate
  • Mac/Linux: source .venv/bin/activate

Navigate into your top-level project folder by typing:

cd myproject

Start the interactive console to interact with your database by typing:

py manage.py shell

2 Exploring the Django ORM

📝 Note: Django uses an ORM (Object Relational Mapping), which acts as an intermediary so you can execute Python code to carry out actions on your database tables.

To interact with the model you created in Chapter 3, you first need to import it into the shell. Type: 

from posts.models import Post

3 Creating and Saving Data

Follow these steps in the shell to create a database entry:

  • Create a new instance of your post model: p = Post()
  • Assign a title to this instance: p.title = 'my first post!'
  • Save this post instance into your database by running the method: p.save()
  • To retrieve all posts saved in your database, type: Post.objects.all()

It will return a QuerySet displaying something generic like <QuerySet [<Post: Post object (1)>]>.

Exit the interactive shell by typing exit().

4 Customizing Model Representation

Because the generic Post object (1) string is not very readable, you can customize how the model is represented.

Open your models.py file inside the posts app. Inside your Post class, create a string method:

    def __str__(self):
        return self.title
💡 Tip: Save the file. You do not need to make a migration for this change because it does not alter the actual database schema, just how the data is displayed in Python.

5 Testing the Updates

Open the terminal and start the shell again using py manage.py shell.

Import the model once more:

from posts.models import Post

Create another new post to test:

p = Post()
p.title = 'this is my second post'
p.save()

Run Post.objects.all() to see the contents of your database. You should now receive a QuerySet that displays the actual text titles you assigned to the posts (e.g., 'my first post!' and 'this is my second post').

Type exit() to leave the shell once you are finished.
