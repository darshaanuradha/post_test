Django Guide: Forms, Views, and Foreign Keys

1. Creating a Custom Form

  • Open your posts app directory and create a new file named forms.py.
  • Import Django's forms module (from django import forms) and import your models (from . import models).
  • Create a new class called CreatePost that inherits from forms.ModelForm.
  • Inside this class, define a nested class named Meta.
  • Within the Meta class, designate the target model and provide a list of fields.
from django import forms
from . import models

class CreatePost(forms.ModelForm):
    class Meta:
        model = models.Post
        fields = ['title', 'body', 'slug', 'banner']
Note: The date is excluded from the fields list because it is auto-generated.

3. Building the Form in the Template

  • Open your post_new.html template and add an HTML <form> element.
  • Set the action attribute to dynamically submit to the named URL (action="{% url 'posts:new-post' %}").
  • Set the method to "POST". Because this form will handle an image upload (the banner), you must also include the attribute enctype="multipart/form-data".
  • Inside the form, put your security tag {% csrf_token %}, output your dynamic form fields using {{ form }}, and include a submit button.
<form action="{% url 'posts:new-post' %}" method="POST" enctype="multipart/form-data">
    {% csrf_token %}
    {{ form }}
    <button type="submit">Submit</button>
</form>

4. Adding an Author Foreign Key to the Model

  • To track which user creates a post, open models.py in your posts app and import the built-in user model at the top: from django.contrib.auth.models import User.
  • Add an author field to your Post model by creating a relational link. The CASCADE rule ensures that if a user is deleted, their corresponding posts are deleted along with them. 
author = models.ForeignKey(User, on_delete=models.CASCADE, default=None)
Important: Because you altered the model, you must go into the Django admin panel (/admin) and delete all of your existing posts first. Otherwise, you will get an error since those old posts lack an assigned author.

Stop the server and apply your new changes to the database by running:

py manage.py makemigrations
py manage.py migrate

5. Saving the Post with the Current User

  • Go back to the post_new function in your posts app's views.py.
  • Create an if block to check if the form is being submitted via POST.
  • Instantiate the form and fill it with the submitted text data and image files.
  • Check if the submitted data is valid. If it is, delay saving to assign the logged-in user, then commit it to the database.
def post_new(request):
    if request.method == 'POST':
        form = forms.CreatePost(request.POST, request.FILES)
        if form.is_valid():
            newpost = form.save(commit=False)
            newpost.author = request.user
            newpost.save()
            return redirect('posts:list')
    else:
        form = forms.CreatePost()
        
    return render(request, 'posts/post_new.html', {'form': form})
