Chapter 3: Django Models & Migrations

Your beginner-friendly guide to setting up databases in Django.

1 Environment Setup

Open your terminal in VS Code and activate your virtual environment by typing source .venv/scripts/activate (Windows) or source .venv/bin/activate (Mac/Linux).

Navigate into your project folder by typing:

cd myproject

2 Creating a Database Model

📝 Note: Models in Django are Python classes that map to tables in your database.

Open the models.py file inside your posts app directory.

Create a new class named Post that inherits from models.Model:

class Post(models.Model):
    # Fields will go here

3 Defining Model Fields

Inside your Post class, define the fields (columns) you want your database table to have.

  • Title: Set title = models.CharField(max_length=75).
  • Body: Set body = models.TextField(). Unlike a CharField, a TextField will relate to a larger text area form input later on.
  • Slug: Set slug = models.SlugField(). A slug is a URL-friendly string (e.g., my-first-post) used to identify the post in the web address.
  • Date: Set date = models.DateTimeField(auto_now_add=True). The auto_now_add=True argument automatically adds a timestamp whenever a new post is saved.

Don't forget to save your file!

4 Applying Built-in Migrations

Django comes with built-in models (like users and administration features) that need to be set up first.

In your terminal, apply these initial built-in migrations by running:

py manage.py migrate

5 Making Migrations for Your App

Now you need to tell Django that you've created a new model. Create a migration file by running:

py manage.py makemigrations

This will look at the Post class you created and generate a new Python file in the migrations folder containing the instructions to build the table.

6 Applying Your New Migration

Finally, execute those instructions and build the table in your database by running the migrate command one more time:

py manage.py migrate
